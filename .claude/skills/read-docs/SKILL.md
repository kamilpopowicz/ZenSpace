# Skill: read-docs

**Cel:**
Zrozumienie aktualnego stanu projektu na podstawie dokumentacji głównej.

**Logika:**
- Czyta (stronicując, jeśli trzeba): `docs/PRD.md`, `docs/SDD.md`, `docs/TECH_STACK.md`, `docs/ROADMAP.md`, `docs/INSTRUCTIONS.md`, `.claude/project-memory.md`.
- Tworzy na tej podstawie krótkie `docs_summary` (1-2 ekrany tekstu) zawierające najważniejsze constrainty (wymagania, ograniczenia) i kryteria akceptacji.
